1. Для чего используются диаграммы объектов? Что на них изображается?
   
Диаграммы объектов используются для отображения снимка (среза) системы в конкретный момент времени. Они показывают:
Конкретные экземпляры классов (объекты)
Текущие значения атрибутов этих объектов
Связи (экземпляры ассоциаций) между объектами
Если диаграмма классов показывает структуру (чертёж), то диаграмма объектов — реальность (фактические объекты и их связи).

2. Как изображают экземпляр класса и экземпляр ассоциации на диаграммах UML?

Экземпляр класса (объект): изображается прямоугольником с двумя секциями:
Верхняя секция: имяОбъекта : ИмяКласса (подчёркивается)
Нижняя секция: атрибуты в формате имя = значение
Пример: лев1 : Лев и ниже возраст = 5
Экземпляр ассоциации (связь): изображается сплошной линией между двумя объектами, может подписываться именем ассоциации и ролями (также подчёркиваются)

3. Как вы думаете, насколько часто приходится создавать диаграммы объектов в разных проектах? Почему?

Диаграммы объектов создаются относительно редко по сравнению с диаграммами классов. Причины:
Они используются в основном для анализа, тестирования и документирования конкретных сценариев
Многие инструменты генерируют их автоматически из отладчика или кода
Наиболее полезны при уточнении требований или объяснении сложных граничных случаев
В простых проектах разработчики часто обходятся без них, обращаясь напрямую к коду или диаграмме классов

class Animal(ABC):
    def __init__(self, name: str):
        self.name = name
    
    
    def make_sound(self) -> None:
        pass

class Lion(Animal):
    def make_sound(self) -> None:
        print(f"{self.name} the lion roars: ROAR!")

class Elephant(Animal):
    def make_sound(self) -> None:
        print(f"{self.name} the elephant trumpets: BRRRAAAH!")

class Raccoon(Animal):
    def make_sound(self) -> None:
        print(f"{self.name} the raccoon screeches: GROWL!")

class Enclosure:
    def __init__(self, number: int, max_capacity: int):
        self.number = number
        self.max_capacity = max_capacity
        self.animals: List[Animal] = []
    
    def add_animal(self, animal: Animal) -> bool:
        if len(self.animals) < self.max_capacity:
            self.animals.append(animal)
            return True
        return False
    
    def remove_animal(self, animal: Animal) -> bool:
        if animal in self.animals:
            self.animals.remove(animal)
            return True
        return False
    
    def has_space(self) -> bool:
        return len(self.animals) < self.max_capacity

class Zookeeper:
    def __init__(self, name: str):
        self.name = name
        self.assigned_enclosures: List[Enclosure] = []
    
    def assign_enclosure(self, enclosure: Enclosure) -> None:
        self.assigned_enclosures.append(enclosure)
    
    def work_elsewhere(self) -> None:
        print(f"{self.name} leaves to work elsewhere after zoo closes")

class Zoo:
    def __init__(self):
        self.enclosures: List[Enclosure] = []
        self.zookeepers: List[Zookeeper] = []
    
    def close(self) -> None:
        print("\n=== ZOO CLOSING ===")
        for keeper in self.zookeepers:
            keeper.work_elsewhere()
        print("The zoo is now closed for the day.")

# Example usage
if __name__ == "__main__":
    # Create zoo
    zoo = Zoo()
    
    # Create enclosures
    enclosure1 = Enclosure(1, 2)
    enclosure2 = Enclosure(2, 3)
    zoo.enclosures.extend([enclosure1, enclosure2])
    
    # Create animals
    lion = Lion("Simba")
    elephant = Elephant("Doon")
    Raccoon = Raccoon("Tsitsi")
    
    # Add animals to enclosures
    enclosure1.add_animal(lion)
    enclosure1.add_animal(elephant)
    enclosure2.add_animal(Raccoon)
    
    # Create and assign zookeepers
    yuri = Zookeeper("Yuri")
    yuri.assign_enclosure(enclosure1)
    zoo.zookeepers.append(Yuri)
    
    # Make sounds
    lion.make_sound()
    elephant.make_sound()
    raccoon.make_sound()
    
    # Close zoo
    zoo.close()
